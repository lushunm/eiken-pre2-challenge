# 問題・語彙データの追加フロー

## 手順

1. **対象ファイルの特定**: 追加するカテゴリの最新パートファイル（`data/*_p*.js` の番号が最大のもの）を開く。
   既存項目のフォーマット・表記規則（ふりがな等）を必ず確認してから作問する。
2. **末尾に追加**: 配列の閉じ括弧 `];` の直前に新項目を追加する。**既存行には一切触れない**
   （ID は並び順から自動付与されるため、途中挿入・削除・並び替えは localStorage の進捗を壊す）。
   - ファイルが大きくなりすぎる場合は新パートファイル（`*_pN.js`）を作成し、
     index.html の `<script>` 一覧と該当アグリゲータ（`data/vocab.js` 等の concat）に登録する。
3. **検査**: `powershell -NoProfile -ExecutionPolicy Bypass -File scripts/check-data.ps1` を実行し PASS を確認。
   NG が出たら追加方法を修正する（末尾追加になっているか、ID 重複がないか）。
4. **機械検証**: `.claude/launch.json` の eiken-pre2 サーバー（ポート 8543）を preview_start で起動し、
   preview_eval で `startRound(対象モード)` → 全問回答 → 結果画面到達を確認する。
   コンソールエラー（preview_console_logs）もチェックする。
5. **コミット & push**: 変更内容（カテゴリ・件数）が分かる日本語メッセージでコミットし、main に push。
   GitHub Pages には 1〜2 分で反映される。CI（data-check）の結果も確認する。
6. **報告**: 追加したカテゴリ・件数・配置ファイルを報告する。

## 作問時の注意

- 問題はオリジナル作問（過去問の転載禁止）。出題形式は 2024 年度リニューアル後の公式形式に準拠。
- 語彙レベルは CEFR-J A2〜B1（準2級相当）の範囲に収める。
- 小学生向け: 和訳・解説の漢字にはふりがな（例:「能力（のうりょく）」）またはひらがな。
- 大量追加（50 件以上）はカテゴリごとにサブエージェントへ分担させてよい。
  ただし**最終的な追記・検査・コミットは直列で行う**（同一ファイルへの並行書き込み禁止）。
